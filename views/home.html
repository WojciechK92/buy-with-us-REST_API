Buy with us - REST API

Short description

This project is a simple REST API designed for an online store. An unauthorized user, by sending a request to a specific endpoint, receives information about a selected product or a list of products. A user who verifies their identity gains the ability to add new products as well as edit and delete products if they were added by them. The project allows authentication using an accessToken and refreshToken.

Routes

This HTTP methods are supported.

Endpoint Method Action
/items/:id GET Fetch item
/items GET Fetch item list
/register POST Register
/login POST Login
/refresh-token POST Refresh Access Token
/items POST Add item
/items/:id PUT Edit item
/items/:id DELETE Delete item
/logout POST Logout

Usage examples

You can use "Postman" to test your application.

  • Configuration

    To make testing easier, it is recommended to initially configure the environment and systematically create collections of requests. The end result of this process can be seen in the attached image (this step is optional, but highly recommended).

  • Registration

    To register a user, use the ‘/register’ endpoint with the POST method by sending the registration data: email and password in the body of the request. In response, you will receive the user’s id, email, and hashed password.

  • Logging in

    To log in a user, use the ‘/login’ endpoint with the POST method by sending the login data: email and password in the body of the request. In response, you will receive the user’s id, email, access token, and refresh token.

  • Fetching a list of items

    Use the ‘/items’ endpoint with the GET method. The response will return a list of items.

  • Adding an item

    Use the ‘/items’ endpoint with the POST method by sending 4 values in the body of the request: name, category, amount, description. During the creation of a new item, validation is performed. For the ‘category’ value, possible variants are: ‘motorization’, ‘real-estate’, ‘house-and-garden’, ‘services’, ‘for-children’, ‘rtv-agd’. To authenticate, it is necessary to send an access token in the header (bearer token). In response, you will receive data of the created item.

  • Editing an item

    Use the ‘/items/:id’ endpoint with the PUT method by sending 4 values in the body of the request: name, category, amount, description. During the edition of item, validation is performed. For the ‘category’ value, possible variants are: ‘motorization’, ‘real-estate’, ‘house-and-garden’, ‘services’, ‘for-children’, ‘rtv-agd’. To authenticate, it is necessary to send an access token in the header (bearer token). In response, you will receive data of the edited item or an error if the item was not created by you.

  • Deleting an item

    Use the ‘/items/:id’ endpoint with the DELETE method. To authenticate, it is necessary to send an access token in the header (bearer token). In response, you will receive a status confirming the deletion of the item or an error if the item was not created by you.

  • Refreshing a token

    Use the ‘/refresh-token’ endpoint with the POST method by sending the refresh token obtained during login in the body of the request. In response, you will receive the user’s data along with a new access token.

  • Logging out

    Use the ‘/logout’ endpoint with the POST method by sending the refresh token obtained during login in the body of the request. In response, you will receive a status confirming the logout.
